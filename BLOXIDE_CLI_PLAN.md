# Bloxide-CLI Implementation Plan

## Overview

Create a `bloxide-codegen` library/binary + `cargo-blox` CLI that moves schema-driven macro expansion from proc-macro time to build-generation time. Proc macros that contain arbitrary Rust code (`transitions!`, `BloxCtx`, `delegatable`) remain as macros. Data-driven macros (`blox_messages!`, `event!`, `StateTopology`, `mailboxes_impls!`) move to TOML-driven codegen.

### Module: `messages.rs` — Generate Message Enums

Input TOML (in `crates/messages/counter-messages/blox.toml`):
```toml
[[messages]]
name = "CounterMsg"
visibility = "pub"

[[messages.variants]]
name = "Tick"

[[messages.variants]]
name = "Reset"
```

Generated output (identical to current `blox_messages!` macro output):
```rust
// Auto-generated by bloxide-codegen. Do not edit manually.

#[derive(Debug, Clone, Copy)]
pub struct Tick;

#[derive(Debug, Clone, Copy)]
pub struct Reset;

#[derive(Debug, Clone, Copy)]
pub enum CounterMsg {
    Tick(Tick),
    Reset(Reset),
}

impl CounterMsg {
    pub fn message_name(&self) -> &'static str {
        match self {
            CounterMsg::Tick(..) => "Tick",
            CounterMsg::Reset(..) => "Reset",
        }
    }
}
```

Implementation: Parse `MessageEnumConfig`, generate `quote!` tokens, format with `prettyplease`, write to file.

**Key rule:** Empty field list `[]` generates a unit struct wrapper (e.g., `Tick` not `Tick {}`). This matches current macro behavior for `Resume {}` producing `pub struct Resume;`.

#[repr(u8)]
pub enum CounterState {
    Ready = 0,
    Done = 1,
}

impl bloxide_core::spec::StateTopology for CounterState {
    const STATE_COUNT: usize = 2;

    fn parent(self) -> Option<Self> {
        match self { _ => None }
    }

    fn is_leaf(self) -> bool {
        match self { _ => true }
    }

    fn path(self) -> &'static [Self] {
        match self {
            CounterState::Ready => &[],
            CounterState::Done => &[],
        }
    }

    fn as_index(self) -> usize {
        match self {
            CounterState::Ready => 0,
            CounterState::Done => 1,
        }
    }
}

#[macro_export]
macro_rules! counter_state_handler_table {
    ($ty:ty) => {
        &[
            &<$ty>::READY_FNS,
            &<$ty>::DONE_FNS,

### Module: `lib.rs` — Public API

```rust
pub mod schema;
mod messages;
mod events;
mod topology;
mod mailboxes;
mod wiring;

use schema::BloxConfig;
use std::path::Path;

pub fn generate_all(config: &BloxConfig, crate_name: &str) -> anyhow::Result<Vec<(String, String)>> {
    let mut files = Vec::new();
    
    if let Some(messages) = &config.messages {
        for msg in messages {
            let code = messages::generate(msg)?;
            files.push((format!("messages_{}.rs", msg.name.to_lowercase()), code));
        }
    }
    
    if let Some(event) = &config.event {
        let code = events::generate(event)?;
        files.push(("events.rs".to_string(), code));
    }
    
    if let Some(topology) = &config.topology {
        let code = topology::generate(topology, crate_name)?;
        files.push(("topology.rs".to_string(), code));
    }
    
    if let Some(wiring) = &config.wiring {
        let code = wiring::generate(wiring)?;
        files.push(("wiring.rs".to_string(), code));
    }
    
    Ok(files)
}

pub fn generate_from_toml(input_path: &Path) -> anyhow::Result<Vec<(String, String)>> {
    let content = std::fs::read_to_string(input_path)?;
    let config: BloxConfig = toml::from_str(&content)?;
    
    // Infer crate name from parent directory
    let crate_name = input_path
        .parent()
        .and_then(|p| p.file_name())
        .and_then(|n| n.to_str())
        .unwrap_or("unknown")
        .to_string();
    
    generate_all(&config, &crate_name)
}
```

### Binary: `main.rs`

```rust
use std::path::{Path, PathBuf};
use bloxide_codegen::generate_from_toml;

fn main() -> anyhow::Result<()> {
    let args: Vec<String> = std::env::args().collect();
    
    // Usage: bloxide-codegen <input-toml> <output-dir>
    if args.len() < 3 {
        eprintln!("Usage: bloxide-codegen <input-toml> <output-dir>");
        std::process::exit(1);
    }
    
    let input = Path::new(&args[1]);
    let output = Path::new(&args[2]);
    
    std::fs::create_dir_all(output)?;
    
    let files = generate_from_toml(input)?;
    for (filename, content) in files {
        let path = output.join(&filename);
        std::fs::write(&path, content)?;
        eprintln!("Generated: {}", path.display());
    }
    
    Ok(())
}
```

### Module: `main.rs`

```rust
use clap::{Parser, Subcommand};

#[derive(Parser)]
#[command(name = "cargo")]
#[command(bin_name = "cargo")]
struct CargoCli {
    #[command(subcommand)]
    command: BloxCommand,
}

#[derive(Subcommand)]
enum BloxCommand {
    #[command(name = "blox")]
    Blox(BloxArgs),
}

#[derive(Parser)]
struct BloxArgs {
    #[command(subcommand)]
    command: BloxSubcommand,
}

#[derive(Subcommand)]
enum BloxSubcommand {
    /// Generate code from all blox.toml files in workspace
    Generate {
        #[arg(long)]
        workspace: Option<PathBuf>,
    },
    /// Generate, then build
    Build {
        #[command(flatten)]
        cargo: clap_cargo::Features,
        #[arg(trailing_var_arg = true)]
        args: Vec<String>,
    },
    /// Generate, then check
    Check {
        #[command(flatten)]
        cargo: clap_cargo::Features,
        #[arg(trailing_var_arg = true)]
        args: Vec<String>,
    },
    /// Generate, then test
    Test {
        #[command(flatten)]
        cargo: clap_cargo::Features,
        #[arg(trailing_var_arg = true)]
        args: Vec<String>,
    },
    /// Generate, then run
    Run {
        #[command(flatten)]
        cargo: clap_cargo::Features,
        #[arg(trailing_var_arg = true)]
        args: Vec<String>,
    },
    /// Watch and regenerate on changes
    Watch {
        #[command(flatten)]
        cargo: clap_cargo::Features,
    },
    /// Scaffold a new blox
    New {
        name: String,
    },
    /// Run spec-to-code lint checks
    Lint,
    /// Run full CI feature matrix
    Ci,
}

fn main() -> anyhow::Result<()> {
    let cli = CargoCli::parse();
    match cli.command {
        BloxCommand::Blox(args) => match args.command {
            BloxSubcommand::Generate { workspace } => generate(workspace),
            BloxSubcommand::Build { cargo, args } => build(cargo, args),
            BloxSubcommand::Check { cargo, args } => check(cargo, args),
            BloxSubcommand::Test { cargo, args } => test(cargo, args),
            BloxSubcommand::Run { cargo, args } => run(cargo, args),
            BloxSubcommand::Watch { cargo } => watch(cargo),
            BloxSubcommand::New { name } => new_blox(name),
            BloxSubcommand::Lint => lint(),
            BloxSubcommand::Ci => ci(),
        },
    }
}
```

### Module: `generate.rs`

```rust
use std::path::{Path, PathBuf};
use walkdir::WalkDir;
use bloxide_codegen::generate_from_toml;

pub fn generate(workspace: Option<PathBuf>) -> anyhow::Result<()> {
    let root = workspace.unwrap_or_else(|| {
        PathBuf::from(std::env::var("CARGO_MANIFEST_DIR").unwrap_or_else(|_| ".".to_string()))
    });
    
    let mut count = 0;
    for entry in WalkDir::new(&root)
        .max_depth(4)
        .into_iter()
        .filter_map(|e| e.ok())
        .filter(|e| e.file_name() == "blox.toml")
    {
        let toml_path = entry.path();
        let crate_dir = toml_path.parent().unwrap();
        let src_dir = crate_dir.join("src");
        let generated_dir = src_dir.join("generated");
        
        std::fs::create_dir_all(&generated_dir)?;
        
        let files = generate_from_toml(toml_path)?;
        for (filename, content) in files {
            let path = generated_dir.join(&filename);
            
            // Only write if changed (preserves mtime for caching)
            let needs_write = if path.exists() {
                std::fs::read_to_string(&path)? != content
            } else {
                true
            };
            
            if needs_write {
                std::fs::write(&path, content)?;
                println!("bloxide: generated {}", path.display());
            }
        }
        
        // Ensure lib.rs includes the generated module
        ensure_generated_mod(&src_dir)?;
        count += 1;
    }
    
    println!("bloxide: processed {} blox.toml files", count);
    Ok(())
}

fn ensure_generated_mod(src_dir: &Path) -> anyhow::Result<()> {
    let lib_rs = src_dir.join("lib.rs");
    let mod_line = "pub mod generated;\n";
    
    if lib_rs.exists() {
        let content = std::fs::read_to_string(&lib_rs)?;
        if !content.contains("mod generated") {
            std::fs::write(&lib_rs, format!("{}\n{}", mod_line, content))?;
        }
    }
    
    // Create generated/mod.rs if not exists
    let generated_mod = src_dir.join("generated").join("mod.rs");
    if !generated_mod.exists() {
        let mod_content = "// Auto-generated module.\n// Files in this directory are generated by bloxide-codegen.\n";
        std::fs::write(&generated_mod, mod_content)?;
    }
    
    Ok(())
}
```

### Module: `forward.rs`

```rust
use std::process::Command;

pub fn forward_to_cargo(cmd: &str, extra_args: &[String]) -> anyhow::Result<()> {
    let cargo = std::env::var("CARGO").unwrap_or_else(|_| "cargo".to_string());
    
    let mut command = Command::new(&cargo);
    command.arg(cmd);
    
    for arg in extra_args {
        command.arg(arg);
    }
    
    let status = command.status()?;
    if !status.success() {
        std::process::exit(status.code().unwrap_or(1));
    }
    Ok(())
}
```

### Module: `build.rs`, `check.rs`, `test.rs`, `run.rs`

Each runs `generate(None)` first, then `forward_to_cargo("build", &args)` etc.

### Module: `watch.rs`

Uses `notify` crate to watch for `blox.toml` changes, regenerates, then runs `cargo check`.

### Module: `new.rs`

Scaffolds a new blox crate following the five-layer pattern:
1. Create `spec/bloxes/<name>.md`
2. Create `crates/messages/<name>-messages/` with `blox.toml` + `src/lib.rs`
3. Create `crates/actions/<name>-actions/` with traits
4. Create `crates/bloxes/<name>/` with `blox.toml`, `src/lib.rs`, `src/ctx.rs`, `src/events.rs`, `src/spec.rs`, `src/tests.rs`
5. Update root `Cargo.toml` workspace members and workspace.dependencies

### Module: `ci.rs`

Runs the full feature matrix:
```rust
pub fn ci() -> anyhow::Result<()> {
    let checks = vec![
        ("bloxide-core", vec!["--no-default-features"]),
        ("bloxide-core", vec!["--no-default-features", "--features", "alloc"]),
        ("bloxide-core", vec!["--features", "std"]),
        ("bloxide-timer", vec!["--target", "riscv32imc-unknown-none-elf"]),
        // ... etc
    ];
    
    for (pkg, args) in checks {
        let status = Command::new("cargo")
            .args(["check", "-p", pkg])
            .args(args)
            .status()?;
        if !status.success() {
            anyhow::bail!("CI check failed for {}", pkg);
        }
    }
    Ok(())
}
```

### Step 1: Migrate `counter-messages`

4. Keep the old macro code behind a feature gate temporarily:

### Step 3: Feature Gate Strategy

All migrated crates get a `codegen` feature (default enabled):
```toml
[features]
default = ["codegen"]
codegen = []
```

In `lib.rs`:
```rust
#[cfg(feature = "codegen")]
pub mod generated;

#[cfg(feature = "codegen")]
pub use generated::*;

#[cfg(not(feature = "codegen"))]
// Keep old macro-based code as fallback
```

This allows gradual migration and rollback.

## Phase 4: Testing & Validation

### Tests for `bloxide-codegen`

1. **Roundtrip tests:** Parse TOML → generate → the generated code should compile.
2. **Golden tests:** Check generated output against expected snapshots.
3. **Equivalence tests:** Generated code must be byte-for-byte identical in behavior to macro-generated code.

### Tests for `cargo-blox`

1. **Integration test:** Run `cargo blox generate` on a temp workspace with `blox.toml`, verify `generated/` files exist.
2. **Stale detection test:** Re-run generate without changes — should not modify mtime.
3. **CI test:** `cargo blox ci` should run all checks successfully.

### Validation of migrated crates

After migration, verify:
- `cargo check -p counter-blox` passes
- `cargo test -p counter-blox` passes  
- `cargo run --example tokio-minimal-demo` passes
- The generated files are committed to git
- The old macro code (behind `#[cfg(not(feature = "codegen"))]`) still compiles
