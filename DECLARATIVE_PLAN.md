# Plan: Everything Declarative

## Goal

Eliminate all hand-written state topology and transition code. Everything — state enums, topology impls, transition rules, guard conditions, action wiring, entry/exit handlers — must be declared in `blox.toml` and generated by `cargo blox generate`. No hand-editing of generated files. Remove the now-dead `#[derive(StateTopology)]` proc macro.

## Context

The codegen (`bloxide-codegen`) already supports the full declarative pipeline:
- `[[topology.states]]` → state enum + `impl StateTopology` (in `generated/topology.rs`)
- `[[topology.transitions]]` → `StateRule { event_tag, matches, actions, guard }` (in `generated/spec_skeleton.rs`)
- `[[topology.entry]]` / `[[topology.exit]]` → `on_entry` / `on_exit` function lists
- Guards with `condition` (Rust expression string) + `target` (`stay`/`reset`/`fail`/`StateName`)
- Actions as function path strings with `{R}` placeholder replacement
- Feature gating via `feature` field on transitions

The `#[derive(StateTopology)]` proc macro in `crates/bloxide-macros/src/state_topology.rs` is a leftover from before codegen existed. It has **zero users** — no crate in the workspace uses it. All generated `topology.rs` files emit the impl directly; test fixtures hand-implement `StateTopology`.

## Audit Findings (pre-work)

- All 7 blox TOML files already have `[[topology.transitions]]` declared (supervisor, ping, pong, counter, worker, pool, bhsm-tst).
- Running `cargo blox generate` produces a diff in **one file**: `crates/bloxide-supervisor/src/generated/spec_skeleton.rs`.
  - The diff removes a `record_killed` transition (lines for `ChildLifecycleEvent::Killed { .. }` → `record_killed::<R>`).
  - This transition exists in the **generated file** but is **missing from `blox.toml`** — it was hand-added to the generated output.
  - The `record_killed` action function exists in `crates/bloxide-supervisor/src/actions.rs` (line 102) and is re-exported from `lib.rs`.
- Other diffs from `cargo blox generate` are pure formatting/import-ordering (no semantic changes).
- `#[derive(StateTopology)]` has zero users in the entire workspace.
