# Blox Spec: `Ping`

## Purpose

The Ping actor initiates and drives a ping-pong exchange with the Pong actor. It demonstrates:

- **Nested composite state**: `Operating(Active, Paused)` ‚Äî three-level deep hierarchy.
- **Internally-driven pause**: After `PAUSE_AT_ROUND`, `Active` itself transitions to `Paused` (no external signal). `Paused::on_entry` calls `set_timer()` (from `bloxide-timer`) to schedule a `PingPongMsg::Resume` delivery.
- **Timer-driven events**: The `Resume` event is generated by a one-shot timer owned by the Ping blox itself ‚Äî not by any external actor or bootstrap task.
- **Event bubbling**: `Paused` delegates unrecognised events (e.g. stray `Pong`) to `Operating` via implicit bubble.
- **Transition-action split**: the first startup ping is sent from `Active::on_entry`; subsequent pings are sent by transition actions before re-entering `Active`.
- **Runtime-managed lifecycle**: `Done` is a terminal state. The runtime detects this via `is_terminal()` and notifies the supervisor automatically ‚Äî no actor code required.
- **Error state**: `Error` is a dedicated fault state. The runtime detects this via `is_error()` and reports `ChildLifecycleEvent::Failed` to the supervisor ‚Äî distinct from normal completion (`Done`).

## Crate Location

- Blox crate: `crates/bloxes/ping/`
- Messages crate: `crates/messages/ping-pong-messages/`
- Actions crate: `crates/actions/ping-pong-actions/`

## State Hierarchy

```mermaid
stateDiagram-v2
    [*] --> Active : runtime calls machine.start()

    state Operating {
        Active
        Paused
    }

    Active --> Active : "PingPongMsg::Pong [!failed and round < MAX_ROUNDS and round != PAUSE_AT_ROUND]"
    Active --> Paused : "PingPongMsg::Pong [round == PAUSE_AT_ROUND]"
    Paused --> Active : "PingPongMsg::Resume (timer-delivered)"
    Active --> Done : "PingPongMsg::Pong [round >= MAX_ROUNDS]"
    Active --> Error : "PingPongMsg::Pong [results.any_failed()]"
```

> `[Init]` is engine-implicit (not in the `PingState` enum). The actor enters Init at construction and waits. The runtime calls `machine.start()` to exit Init and enter the first operational state. The runtime calls `machine.reset()` to exit all operational states and re-enter Init.
> `Operating` is a composite state (never active). `Active`, `Paused`, `Done`, and `Error` are leaf states.
> There is no external `Pause` signal ‚Äî the transition is internally driven by the round counter.
> The `Error` guard (`results.any_failed()`) is checked before round-based conditions, so action failures take priority.

## States

| State | Kind | Description |
|-------|------|-------------|
| `[Init]` | engine-implicit | Waiting for runtime `start()`; `on_init_entry` resets domain state |
| `Operating` | composite | Groups `Active` and `Paused`; absorbs stray `Pong` while in `Paused` |
| `Active` | leaf | One round in progress; sends ping on entry, waits for pong |
| `Paused` | leaf | Exchange suspended; sets a timer on entry; stray `Pong` bubbles to Operating |
| `Done` | leaf, terminal | Terminal state; `is_terminal()` returns `true`; runtime reports `ChildLifecycleEvent::Done` |
| `Error` | leaf, error | Error state; `is_error()` returns `true`; runtime reports `ChildLifecycleEvent::Failed` |

## Events

| Event | Source | Handled by | Reaction | Side effects |
|-------|--------|-----------|----------|--------------|
| `PingPongMsg::Pong(_)` | Pong actor | `Active` | `Transition(Error)` if `results.any_failed()` | action failure takes priority; Error::on_entry fires |
| `PingPongMsg::Pong(_)` | Pong actor | `Active` | `Transition(Done)` if `round >= MAX_ROUNDS` | `Self::forward_ping` runs before the guard; `Done::on_entry` logs completion |
| `PingPongMsg::Pong(_)` | Pong actor | `Active` | `Transition(Paused)` if `round == PAUSE_AT_ROUND` | `Self::forward_ping` runs before the guard; timer set in `Paused::on_entry` |
| `PingPongMsg::Pong(_)` | Pong actor | `Active` | `Transition(Active)` if `round < PAUSE_AT_ROUND` | `Self::forward_ping` runs before the guard; `Active::on_entry` increments/logs the next round |
| `PingPongMsg::Pong(_)` | Pong actor | `Paused` ‚Üí bubbles ‚Üí `Operating` | `Stay` | exchange is paused; pong absorbed by Operating |
| `PingPongMsg::Resume(_)` | Timer (self) | `Paused` | `Transition(Active)` | `Self::forward_ping` sends the resume ping before the transition; `Active::on_entry` then increments/logs the next round |
| any unhandled | ‚Äî | root (no rules) | dropped | none |

Lifecycle control (`start`, `reset`) is handled by the runtime ‚Äî these do not appear as events.

## Context

`PingCtx` uses `#[derive(BloxCtx)]` to generate accessor trait impls and a constructor. It is generic over `B`, a composite behavior type supplied by the wiring binary's impl crate. This keeps the blox crate free of any impl-crate imports (Invariant 9).

```rust
#[derive(BloxCtx)]
pub struct PingCtx<
    R: BloxRuntime,
    B: HasCurrentTimer + CountsRounds,
> {
    #[self_id]
    pub self_id: ActorId,
    #[provides(HasPeerRef<R>)]
    pub peer_ref: ActorRef<PingPongMsg, R>,   // target for pong reply
    #[provides(HasSelfRef<R>)]
    pub self_ref: ActorRef<PingPongMsg, R>,   // target for timer delivery
    #[provides(HasTimerRef<R>)]
    pub timer_ref: ActorRef<TimerCommand, R>, // timer service channel
    #[delegates(HasCurrentTimer, CountsRounds)]
    pub behavior: B,                          // concrete type injected at wiring time
}
```

Behavior traits are delegated automatically by `#[derive(BloxCtx)]` through the
companion `__delegate_*` macros generated by `#[delegatable]` on the action-crate
traits. In this example, `PingCtx` forwards:
- `CountsRounds` ‚Üí `self.behavior.round()` / `self.behavior.set_round()`
- `HasCurrentTimer` ‚Üí `self.behavior.current_timer()` / `self.behavior.set_current_timer()`

The binary injects a concrete `B` (e.g., `PingBehavior` from `embassy-demo-impl`) at construction time. Tests supply a local `TestBehavior` struct. The blox crate imports neither.

No `supervisor_ref` ‚Äî actors don't hold a reference to their supervisor.

## Message Contracts

Both Ping and Pong share the `PingPongMsg` enum from `ping-pong-messages`.

### Receives (`PingPongMsg`)

| Variant | Payload | Source |
|---------|---------|--------|
| `PingPongMsg::Pong(Pong { round })` | round number | Pong actor |
| `PingPongMsg::Resume(Resume)` | none | Timer task (scheduled by `set_timer()` in `Paused::on_entry`) |

### Sends

| Target | Message | When |
|--------|---------|------|
| `peer_ref` | `PingPongMsg::Ping(Ping { round: 1 })` | `Active::on_entry` via `send_initial_ping` (round 1 only) |
| `peer_ref` | `PingPongMsg::Ping(Ping { round })` | `Active` transition action `Self::forward_ping` via `send_ping` (rounds 2+) |

The runtime notifies the supervisor of lifecycle events (`Started`, `Done`, `Failed`, `Reset`, `Stopped`, `Alive`) automatically ‚Äî no explicit sends from actor code.

## Entry / Exit Actions

Most action functions are imported from `ping-pong-actions` and `bloxide-log`. The blox `PingSpec` impl block also defines thin local helper methods that adapt action-crate functions to blox-local constants or extract event payloads:

- `Self::log_pong_received` ‚Äî logs a received Pong using `blox_log_debug!`
- `Self::forward_ping` ‚Äî calls `send_ping` from `ping-pong-actions`
- `Self::schedule_pause_timer` ‚Äî calls `schedule_resume(ctx, PAUSE_DURATION_MS)` from `ping-pong-actions`
- `Self::cancel_pause_timer` ‚Äî calls `cancel_current_timer` from `ping-pong-actions`
- `Self::log_round`, `Self::log_done`, `Self::log_error` ‚Äî logging wrappers using `blox_log_info!`

| State | on_entry | on_exit |
|-------|----------|---------|
| `[Init]` (engine) | reset behavior via `B::default()` | ‚Äî |
| `Operating` | ‚Äî | ‚Äî |
| `Active` | `increment_round`, `Self::log_round`, `send_initial_ping` | ‚Äî |
| `Paused` | `Self::schedule_pause_timer` | `Self::cancel_pause_timer` |
| `Done` | `Self::log_done` | ‚Äî |
| `Error` | `Self::log_error` | ‚Äî |

Each entry in the `on_entry`/`on_exit` column is a function with the signature `fn(&mut PingCtx<R, B>)`. Multiple actions are composed by including them in the `&'static [fn(&mut Ctx)]` slice.
`send_initial_ping` only sends on the first entry to `Active` (`round == 1`);
subsequent pings are sent by `Self::forward_ping` transition actions.

## LCA Exit/Entry Examples

### `Active ‚Üí Done` (round >= MAX_ROUNDS)
```
source_path: [Operating, Active]
target_path: [Done]
LCA = None (Operating vs Done are top-level siblings with no common user ancestor)

Exit:   Active.on_exit      ‚Üê (empty)
        Operating.on_exit   ‚Üê (empty)
Entry:  Done.on_entry       ‚Üê log_done (is_terminal() notifies supervisor via runtime)
```

### `Active ‚Üí Paused` (round == PAUSE_AT_ROUND, internally driven)
```
source_path: [Operating, Active]
target_path: [Operating, Paused]
LCA = Operating (index 0)

Exit:   Active.on_exit      ‚Üê (empty)
Entry:  Paused.on_entry     ‚Üê schedule_resume(ctx, PAUSE_DURATION_MS)
```
> `Operating::on_exit` does NOT fire ‚Äî this is an intra-subtree transition.

### `Paused ‚Üí Active` (timer delivers Resume)
```
source_path: [Operating, Paused]
target_path: [Operating, Active]
LCA = Operating (index 0)

Action: Self::forward_ping  ‚Üê sends Ping with the current round before re-entry
Exit:   Paused.on_exit      ‚Üê cancel_current_timer
Entry:  Active.on_entry     ‚Üê round++, log_round, send_initial_ping (no-op because round != 1)
```

### `Active ‚Üí Error` (results.any_failed())
```
source_path: [Operating, Active]
target_path: [Error]
LCA = None (Operating vs Error are top-level siblings with no common user ancestor)

Exit:   Active.on_exit      ‚Üê (empty)
        Operating.on_exit   ‚Üê (empty)
Entry:  Error.on_entry      ‚Üê log_error (is_error() notifies supervisor via runtime as Failed)
```

### `Done ‚Üí [Init]` (runtime calls machine.reset())
```
current_state = Done, source_path = [Done]

Reset exits: Done.on_exit   (no-op)
Then: on_init_entry()       ‚Üê resets behavior via B::default()
Runtime emits: ChildLifecycleEvent::Reset to supervisor
```

## Constants

| Name | Value | Description |
|------|-------|-------------|
| `MAX_ROUNDS` | 5 | Total ping-pong exchanges before Done |
| `PAUSE_AT_ROUND` | 2 | Round at which Active internally pauses itself |
| `PAUSE_DURATION_MS` | 150 | Timer duration for the pause |

## Acceptance Criteria

- [x] `machine.start()` exits Init and enters Operating then Active; `Active::on_entry` fires; `increment_round` and `send_initial_ping` run
- [x] `PingPongMsg::Pong(n)` in `Active` with `round < PAUSE_AT_ROUND` self-transitions
- [x] `PingPongMsg::Pong(n)` in `Active` with `round == PAUSE_AT_ROUND` transitions to `Paused`; timer is set
- [x] After `PAUSE_DURATION_MS`, `PingPongMsg::Resume` arrives at `Paused` via mailbox; transitions to `Active`
- [x] Stray `PingPongMsg::Pong(n)` in `Paused` bubbles to `Operating` ‚Üí `Stay`
- [x] `PingPongMsg::Pong(n)` in `Active` with `round >= MAX_ROUNDS` transitions to `Done`
- [x] `is_terminal(&PingState::Done)` returns `true`
- [x] `is_error(&PingState::Error)` returns `true`; runtime reports `ChildLifecycleEvent::Failed` (not `Done`)
- [x] `PingPongMsg::Pong(n)` in `Active` when `results.any_failed()` transitions to `Error`; the error guard is checked before round-based guards
- [x] `machine.reset()` from any state (including `Error`) exits all states and calls `on_init_entry`; behavior reset via B::default()
- [x] `on_init_entry` does NOT send any messages ‚Äî it only resets domain state
- [x] Unknown events bubble to root (no root rules) and are silently dropped

## Implementation Notes

- `R: BloxRuntime` is the only runtime bound. Timer functionality comes from `bloxide-timer` (the `HasTimerRef<R>` accessor trait and `set_timer()` / `cancel_timer()` action functions).
- `try_send` is used (not `send`) throughout because entry/exit handlers are synchronous `fn` ‚Äî they cannot await.
- `self_ref` in the context is needed because `set_timer` needs a target `ActorRef` to deliver `Resume` to.
- `timer_ref` in the context is the `ActorRef<TimerCommand, R>` channel to the timer service.
- `cancel_pause_timer` is called in `Paused::on_exit` to prevent stale timer deliveries after leaving `Paused`.
- `on_init_entry` resets the behavior type to its default (`ctx.behavior = B::default()`) so all domain counters are clean after every reset.
- The blox crate depends on `ping-pong-actions` (traits + generic functions), `bloxide-log` (feature-gated logging macros), and `bloxide-timer` (for `TimerCommand` and `HasTimerRef<R>`). Concrete types (e.g., `PingBehavior`) live in an impl crate and are injected by the binary; the blox crate never imports any impl crate (Invariant 9).
- `PingSpec` defines local helper methods (`Self::schedule_pause_timer`, `Self::cancel_pause_timer`, `Self::log_round`, `Self::log_done`, `Self::log_error`, `Self::log_pong_received`, `Self::forward_ping`) that adapt action-crate functions to blox-local constants or event payloads. These are thin wrappers ‚Äî all domain logic remains in `ping-pong-actions`.
- See `spec/architecture/08-supervision.md` for how the runtime manages lifecycle.
- See `spec/architecture/12-action-crate-pattern.md` for the layered architecture.
